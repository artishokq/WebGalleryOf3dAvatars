# REPORT - Web-витрина 3D-аватаров

## Суть приложения

**Web-витрина 3D-аватаров** - это одностраничное веб-приложение, которое позволяет пользователю просматривать коллекцию 3D-моделей персонажей, загружать собственные модели в формате `.glb`/`.gltf`, а также настраивать внешний вид и окружение модели в интерактивном 3D-редакторе прямо в браузере.

Приложение состоит из двух основных страниц:

- **Галерея** - отображает карточки встроенных и загруженных пользователем моделей с миниатюрами. Пользователь может добавить свою модель или удалить ранее загруженную.
- **Редактор** - открывает выбранную модель в 3D-сцене с возможностью вращения, панорамирования, зума, смены окружения, управления анимациями, переключения видимости мешей, изменения оттенка скина, создания скриншота и включения пост-эффектов.

Пользовательские модели сохраняются в IndexedDB браузера и не теряются при перезагрузке. Интерфейс поддерживает два языка (русский и английский) и две темы оформления (светлую и тёмную).

## Технологии

### React 19

React - декларативная JavaScript-библиотека для построения пользовательских интерфейсов на основе компонентов. В проекте React используется как основной UI-фреймворк: все страницы, виджеты, фичи и UI-элементы реализованы как React-компоненты. Использована версия 19 с поддержкой нового компилятора и улучшенной моделью конкурентного рендеринга.

**Сравнение с аналогами:** Vue.js предлагает похожий компонентный подход с более встроенной реактивностью, Svelte компилирует компоненты в чистый JS без виртуального DOM. React был выбран за наиболее зрелую экосистему 3D-библиотек (React Three Fiber/Drei), которые отсутствуют у конкурентов.

### TypeScript 5.9

TypeScript - надмножество JavaScript с системой статических типов. В проекте TypeScript используется повсеместно: все компоненты, хуки, хранилища и утилиты строго типизированы. Это позволяет ловить ошибки на этапе компиляции, обеспечивает автодополнение в IDE и служит живой документацией к коду.

**Сравнение с аналогами:** Альтернативой является написание на чистом JavaScript с JSDoc-комментариями или использование Flow от Meta. TypeScript выбран как индустриальный стандарт с наилучшей поддержкой в экосистеме React и Three.js.

### Vite 7

Vite — современный сборщик и dev-сервер для фронтенд-проектов, использующий нативные ES-модули для мгновенного старта в режиме разработки и Rollup для продакшен-сборки. В проекте Vite отвечает за быструю пересборку при изменении файлов (HMR), сборку финального бандла и интеграцию с плагинами (Tailwind CSS, React).

**Сравнение с аналогами:** Webpack - классический сборщик, но значительно медленнее при разработке. Turbopack (Next.js) ориентирован на SSR-фреймворк. Vite выбран за простоту конфигурации, скорость и нативную поддержку TypeScript.

### Tailwind CSS 4

Tailwind CSS - utility-first CSS-фреймворк, позволяющий стилизовать элементы прямо в разметке через утилитарные классы. В проекте Tailwind используется для всей стилизации: адаптивная верстка, тёмная тема, анимации, отзывчивый дизайн.

**Сравнение с аналогами:** Styled-components и CSS Modules предлагают изолированные стили, но требуют отдельных файлов или runtime-JS. Bootstrap предоставляет готовые компоненты, но менее гибок. Tailwind выбран за высокую скорость разработки, отсутствие runtime-нагрузки и отличную интеграцию с Vite 7 через `@tailwindcss/vite`.

### Three.js 0.183

Three.js - основная JavaScript-библиотека для создания и отображения 3D-графики в браузере через WebGL. В проекте Three.js используется как «движок»: загрузка GLTF-моделей, работа с материалами, освещением, камерой, анимацией и скелетными мешами.

**Сравнение с аналогами:** Babylon.js - полноценный 3D-движок с большим API, но тяжелее. PlayCanvas ориентирован на игры. Three.js выбран за лёгкость, минималистичный API и наличие React-биндинга (React Three Fiber).

### React Three Fiber 9

React Three Fiber (R3F) - React-рендерер для Three.js, позволяющий описывать 3D-сцену декларативно через JSX. В проекте R3F оборачивает всю 3D-сцену: `<Canvas>`, источники света, модели, пост-процессинг - всё описано как React-компоненты с реактивным обновлением.

**Сравнение с аналогами:** Можно работать с Three.js напрямую через императивный код, но это усложняет синхронизацию UI-состояния с 3D-сценой. TresJS (для Vue) - аналог R3F, но с меньшей экосистемой. R3F выбран как единственное зрелое решение для React.

### React Three Drei 10

Drei - коллекция готовых хелперов и компонентов для R3F: OrbitControls, Environment (HDRI-окружение), ContactShadows, Html-оверлеи, хуки `useGLTF` и `useAnimations` и многое другое. В проекте Drei используется для загрузки моделей, управления камерой, отображения окружения и теней.

**Сравнение с аналогами:** Без Drei пришлось бы писать обёртки над Three.js вручную. Drei - стандартная companion-библиотека для R3F, аналогов нет.

### React Three Postprocessing 3

React Three Postprocessing - React-обёртка над библиотекой `postprocessing`, обеспечивающая пост-эффекты рендеринга. В проекте используется для эффекта Bloom (свечение), который применяется к 3D-сцене по желанию пользователя.

**Сравнение с аналогами:** Можно реализовать пост-процессинг через Three.js `EffectComposer`, но это требует императивного кода. React Three Postprocessing интегрируется с R3F декларативно.

### Zustand 5

Zustand - минималистичная библиотека управления состоянием для React, основанная на подписках. В проекте Zustand используется для двух хранилищ: состояние редактора аватара (анимация, скин, окружение, тулбар, триггеры) и список пользовательских моделей (с персистом в localStorage).

**Сравнение с аналогами:** Redux - стандарт де-факто, но требует много бойлерплейта (actions, reducers, selectors). MobX использует наблюдаемые объекты, что усложняет отладку. Jotai/Recoil атомарны, но менее удобны для сложного состояния. Zustand выбран за минимализм, отсутствие бойлерплейта и нативную интеграцию с React.

### React Router 7

React Router - стандартная библиотека клиентской маршрутизации для React. В проекте обеспечивает навигацию между страницей галереи (`/`) и редактором модели (`/editor/:modelId`), а также вложенную раскладку с общим шапкой (Layout).

**Сравнение с аналогами:** TanStack Router предлагает type-safe роутинг, но более сложен в настройке. Next.js Router привязан к фреймворку. React Router выбран как проверенное решение для SPA с простым API.

### idb-keyval 6

idb-keyval - минималистичная обёртка над IndexedDB с key-value API. В проекте используется для сохранения загруженных пользователем 3D-моделей (Blob) и миниатюр в IndexedDB, что позволяет хранить файлы размером в десятки мегабайт без ограничений localStorage.

**Сравнение с аналогами:** Dexie.js - более мощная ORM-подобная обёртка над IndexedDB, но избыточна для key-value сценария. Нативный IndexedDB API громоздкий и callback-ориентированный. idb-keyval выбран за минимальный размер (< 1 КБ) и простейший API (`get`/`set`/`del`).

### Feature-Sliced Design (FSD)

FSD - архитектурная методология для фронтенд-проектов, разделяющая код на слои (app -> pages -> widgets -> features -> entities -> shared) с жёсткими правилами импорта: каждый слой может зависеть только от нижележащих. В проекте FSD обеспечивает предсказуемую структуру: 6 слоёв, 12 feature-слайсов, 5 виджетов, 2 сущности.

**Сравнение с аналогами:** «Папки по типу» (components/hooks/utils) - просто, но плохо масштабируется. «Модульная архитектура» (domain-driven) требует больше дизайн-решений. FSD выбран как чёткий стандарт для React-проектов средней сложности.

## Заключение

В ходе работы над проектом удалось создать полноценное SPA для просмотра и настройки 3D-моделей, работающее целиком в браузере без серверной части. Наибольшую сложность представляла работа с 3D-рендерингом: инициализация камеры, нормализация моделей разных размеров, взаимодействие OrbitControls с React-жизненным циклом - всё это потребовало глубокого понимания как Three.js, так и React Three Fiber.

Связка **React + R3F + Drei** оказалась очень мощной, декларативный подход к 3D-сцене позволяет быстро добавлять фичи (смена окружения, пост-эффекты, анимации), не погружаясь в императивный Three.js-код. Однако отладка гонок состояний между React-эффектами и render-loop Three.js - нетривиальная задача, требующая чёткого понимания обоих миров.

Методология **FSD** помогла с самого начала выстроить чёткую структуру, благодаря чему добавление новых фич, например, тулбара или пост-эффектов, не приводило к хаосу импортов. Общее впечатление от стека — он отлично подходит для интерактивных 3D-приложений средней сложности, работающих целиком на клиенте.
